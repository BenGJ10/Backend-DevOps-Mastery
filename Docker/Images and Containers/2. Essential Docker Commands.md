# Essential Docker Commands

## Overview

Docker provides a set of commands that allow developers to **build, run, inspect, manage, and delete containers and images**.

These commands form the foundation of modern backend deployment workflows and are used daily in development, testing, and production environments.

A typical lifecycle looks like:

```
Build Image → Run Container → Monitor → Stop → Remove → Clean Up
```

Mastering these commands ensures predictable deployments and efficient environment management.

---

## Building Images

### What Is Image Building?

Building an image converts your **application + Dockerfile** into a portable runtime artifact that can be executed anywhere Docker is available.

---

### Build an Image

```bash
docker build -t docker-demo-java .
```

**Explanation:**

* `docker build` → creates an image
* `-t` → assigns a readable tag (name)
* `.` → specifies the build context (current directory)

---

### List Images

```bash
docker images
```

OR

```bash
docker image ls
```

Displays:

* repository
* tag
* image ID
* creation time
* size

---

### Remove an Image

```bash
docker rmi docker-demo-java
```

Force removal:

```bash
docker rmi -f docker-demo-java
```

**Note:**
An image cannot be deleted if a container is using it unless forced.

---

## Running Containers

### What Is Container Execution?

Running a container means creating an isolated runtime environment from an image where the application process executes.

---

### Run a Container

```bash
docker run -p 8080:8080 docker-demo-java
```

**Port Mapping Format:**

```
HOST_PORT : CONTAINER_PORT
```

Requests to `localhost:8080` are forwarded to the container.

---

### Run in Detached Mode

```bash
docker run -d -p 8080:8080 docker-demo-java
```

Runs the container in the background.

Detached mode is the standard approach in production systems.

---

### Assign a Container Name

```bash
docker run -d -p 8080:8080 --name spring-app docker-demo-java
```

Naming containers simplifies logging, debugging, and orchestration.

---

## Inspecting Containers

### List Running Containers

```bash
docker ps
```

Shows:

* container ID
* image
* uptime
* ports

---

### List All Containers

```bash
docker ps -a
```

Includes stopped containers.

Useful when troubleshooting failed startups.

---

## Viewing Logs

### Show Container Logs

```bash
docker logs spring-app
```

Displays application output such as server startup messages.

---

### Stream Logs in Real Time

```bash
docker logs -f spring-app
```

Equivalent to a live log tail.

This is commonly used for monitoring backend services.

---

## Executing Commands Inside Containers

### Open an Interactive Shell

```bash
docker exec -it spring-app bash
```

Allows inspection of the container environment.

Typical use cases:

* verifying files
* checking runtime versions
* debugging configuration

Exit using:

```bash
exit
```

---

## Starting and Stopping Containers

### Stop a Running Container

```bash
docker stop spring-app
```

Performs a graceful shutdown by sending a termination signal to the main process.

---

### Start a Stopped Container

```bash
docker start spring-app
```

Reuses the existing container without rebuilding the image.

---

### Restart a Container

```bash
docker restart spring-app
```

Common after configuration changes.

---

## Removing Containers

### Remove a Stopped Container

```bash
docker rm spring-app
```

Containers must be stopped before removal.

---

### Force Remove a Container

```bash
docker rm -f spring-app
```

Stops and deletes the container in one step.

Useful during rapid development cycles.

---

## Cleaning Up Docker Resources

Over time, unused containers and images consume disk space.

---

### Remove Stopped Containers

```bash
docker container prune
```

Deletes all stopped containers.

---

### Remove Unused Images

```bash
docker image prune
```

Clears dangling images that are no longer referenced.

---

### System-Wide Cleanup

```bash
docker system prune -a
```

Removes:

* stopped containers
* unused networks
* dangling images
* build cache

**Caution:** This operation is destructive.

---

## Most Important Commands (Core Set)

These commands cover the majority of real-world backend workflows:

```bash
docker build -t myapp .
docker run -d -p 8080:8080 --name app myapp
docker ps
docker logs -f app
docker exec -it app bash
docker stop app
docker rm app
docker rmi myapp
```

Engineers who understand this lifecycle can operate Docker confidently.

## Interview Questions

### 1. What does `docker build` do?

**Answer:**
It creates a Docker image from a Dockerfile and application code.

---

### 2. What is the difference between `docker run` and `docker start`?

**Answer:**
`docker run` creates and starts a new container, while `docker start` runs an existing stopped container.

---

### 3. Why use detached mode?

**Answer:**
Detached mode allows containers to run in the background without blocking the terminal.

---

### 4. What command is used to view container logs?

**Answer:**
`docker logs`.

---

### 5. Why should containers be removed after use?

**Answer:**
Removing containers prevents resource waste and keeps the environment clean.

---

## Summary

* Docker commands control the full container lifecycle

* Images are built once and reused

* Containers are created, monitored, and replaced

* Logs enable debugging

* Cleanup prevents disk exhaustion
