# Introduction to Git

## Overview

![Git Logo](../images/git.png)

**git** is a **distributed version control system** that records, manages, and preserves the complete history of changes made to a codebase.  

It enables developers to work on the same project concurrently, maintain multiple versions of the code, and safely experiment with changes without risking the stability of the main codebase.

Git operates locally on a developer’s machine and does not rely on a central server, which makes it fast, reliable, and resilient to failures.

---

## What Problem Does Git Solve?

In real-world software development, codebases evolve continuously and are often modified by multiple developers simultaneously.  
Without a version control system, teams face serious challenges such as:

- Loss of previously working code
- Difficulty identifying when and where a bug was introduced
- Inability to collaborate without overwriting each other’s work
- No structured way to revert to a stable version

Git solves these problems by maintaining a **versioned history of the project**, where every meaningful change is recorded, identifiable, and reversible.

---

## What Git Does for Us

### 1. Tracks Code Evolution Precisely

Git stores every logical change as a **commit**, capturing:
- The exact code modifications
- The author of the change
- The time of the change
- The intent behind the change (commit message)

This allows developers to analyze how the system evolved and confidently revert to any previous state if required.

### 2. Enables Isolated Development Using Branches

Git allows developers to create **branches**, which are independent lines of development.  
This enables:
- Feature development without disturbing stable code
- Bug fixes without blocking ongoing work
- Parallel development across teams

Branches make experimentation safe and structured.

### 3. Supports Structured Collaboration

Multiple developers can work on the same project and later **merge** their changes.  
Git detects conflicting changes automatically and provides mechanisms to resolve them explicitly, ensuring no silent overwrites occur.

This is essential in backend systems where multiple services, APIs, and configurations change together.

### 4. Ensures Accountability and Traceability

Every change in Git is traceable to an individual developer and a specific point in time.  
This makes Git invaluable for:
- Code reviews
- Debugging production issues
- Auditing changes in enterprise systems

## Git vs GitHub

Although often used together, Git and GitHub serve different purposes.

### Git

Git is a **version control system** that runs locally and manages the history of a project by recording changes to files over time.  
It allows developers to commit, branch, merge, and revert code independently of any internet connection.

Example:
```bash
git init
git add .
git commit -m "Initial commit"
```

### GitHub

GitHub is a **cloud-based platform** that hosts Git repositories and provides collaboration tooling on top of Git.
It enables teams to share code, review changes, manage issues, and integrate automated workflows such as CI/CD pipelines.

GitHub depends on Git, but Git itself does not depend on GitHub.

### Key Differences

| Aspect              | Git                           | GitHub                                        |
| ------------------- | ----------------------------- | --------------------------------------------- |
| Nature              | Version control system        | Repository hosting and collaboration platform |
| Execution           | Local machine                 | Cloud-based                                   |
| Internet dependency | Not required                  | Required                                      |
| Primary role        | Track and manage code history | Share, review, and collaborate on code        |

   Explain isolated development and how branches enable parallel work without destabilizing the main code.

## Summary

* Git is a distributed version control system that maintains a complete, reliable history of code changes

* It enables safe experimentation, structured collaboration, and traceable development

* Git works locally and does not require a central server

* GitHub is a hosting and collaboration platform built on top of Git

* Mastering Git is essential for professional backend engineering workflows
